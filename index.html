Oque é HTML?

O HTML, conhecido como Linguagem de Marcação de Hipertexto,(HyperText Markup Language) é a base fundamental para construir páginas da web e aplicativos online. Ele usa uma série de marcações para definir a estrutura e o estilo de um documento. Quando falamos de hipertexto, estamos nos referindo a textos que têm links para outros textos, criando uma experiência de navegação conectada. Ao contrário das linguagens de programação, o HTML não tem o poder de criar funcionalidades dinâmicas. Sua função principal é organizar o conteúdo da página, permitindo que os usuários criem seções, parágrafos e links por meio de elementos, tags e atributos. Embora seja essencial para apresentar e organizar informações na web, para funcionalidades mais avançadas e interações dinâmicas, geralmente é combinado com outras tecnologias, como CSS (para estilos) e JavaScript (para interatividade).

A estruta básica do html segue o seguinte padrão:

<!DOCTYPE html>
<html lang="pt-br">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Tags HTML</title>
</head>
<body>
</body>
</html>

<!DOCTYPE html>: Esta é uma declaração do tipo de documento (Document Type Declaration - DTD) para indicar ao navegador que o documento HTML está seguindo a especificação HTML5.

X<html lang="pt-br">: Abre o elemento raiz do documento HTML. O atributo lang é usado para indicar o idioma do documento. Neste caso, "pt-br" representa o português brasileiro.

<head>: Contém metadados sobre o documento, como o conjunto de caracteres utilizado, configurações de viewport (para tornar o site responsivo) e o título da página.

<meta charset="UTF-8">: Especifica que o conjunto de caracteres UTF-8 deve ser usado para interpretar o conteúdo da página. Isso é importante para garantir que caracteres especiais e acentos sejam exibidos corretamente.

<meta name="viewport" content="width=device-width, initial-scale=1.0">: Configurações relacionadas à visualização em dispositivos móveis. width=device-width significa que a largura da tela deve ser igual à largura do dispositivo, e initial-scale=1.0 define o nível de zoom inicial.

<title>Tags HTML: Define o título da página que será exibido na aba do navegador ou na barra de título.

<body>: Contém o conteúdo real da página, como texto, imagens, links, etc. <body> e </body>.

Imagens e direitos autorais


PNG E JPEG

png: fundo transparente, qualidade melhor, imagens menos compactadas mas permite animações e gifs.(recomendo usar esse tipo de arquivo)
jpeg: Não tem fundo transparente; imagens mais compactas, porém, ao dar zoom a resolução diminui.

Imagens dinâmicas

Para colocar imagens dinâmicas, primeiro temos que adicionar a tag PICTURE, que serve, basicamente para poder colocar imagens que se adaptam ao tamanho do dispositivo. Depois coloque uma imagem como tamanho padrão, e veja abaixo o código usado para colocar imagens que se adaptam

Uma coisa importante sobre as imagens que se adaptam sozinha, é que as imagens tem que estar na ordem, da mesma forma que no código a baixo:

<picture>

<source media="(max-width: 700px)" srcset="imagens/fotop.png" type="image/png"> //imagem pequena

<source media="(max-width:1000px)" srcset="imagens/fotom.png" type="image/png"> //imagem média

<img src="imagens/fotog.png" alt="imagem flexível">//imagem grande </picture>

O atributo "max-width" serve para demilitar o máximo de pixels que imagem poderá aparecer no site, se passar, outra imagem aparecerá no lugar

Sons

Para conseguir músicas ou aúdios facilmente, você pode acessar o youtube studio, lá haverá várias opções de aúdios e músicas para você escolher
obs:haverá aúdios e músicas que você terá que "dar os créditos" e outros não.

Agora, para colocar esses sons baixados temos que usar a tag "audio"

<audio src=" controls autoplay></audio>

O 'autoplay', como o nome já sugere, é automático. Isso significa que quando você entra no site, o áudio tocará automaticamente. No entanto, geralmente os navegadores bloqueiam isso.Esse é o jeito mais básico de colocar aúdio no seu site, mas podemos adicionar algumas outras coisas para deixar "mais completo"

Existem 3 principais formatos de aúdio, que são eles o mp3, hav e o ogg

na tag "audio" tem o atríbuto "preload" que define se o aúdio será baixado ou não quando o usuário entrar no site.O mais recomendado segundo o guanabara é o metadata, que usando ele, baixará as infomações básicas sobre o aúdio, e só quando o usuário apertar para rodar o aúdio que ele será baixado por completo. Existem ainda mais duas opções que são o auto que baixará o aúdio assim que o usuário entrar no site(não recomendado pois o seu site só terá sido carregado 100% quando o aúdio também ser carragado) e o none, que não carrega nada.

<audio preload="metadata" autoplay controls loop muted(não muito recomendado)>

<source src="audio.mp3" type="audio/mpeg">

<source src="audio.ogg" type="audio/ogg">

<source src="audio.wav" type="audio/wav">

Acabei de perceber que o Google Chrome não suporta mp3 e sim o formato wav, por isso é importante colocar vários formatos de áudio(guanabara acabou de falar que wav não é recomendado porque é muito pesado, aí eu coloquei no formato ogg e foi)
